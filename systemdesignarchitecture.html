Event Flows

01Marketer enters goal
User submits marketing objective via App Service API
App Service · POST /api/v1/agents/generate-campaign
02App creates AgentRun record
Stores run_type='campaign_generation' with input goal
App Service · system.agent_runs
03App calls Agent Service (HTTP sync)
Forwards goal to Agent Service for LangGraph execution
HTTP · POST /api/v1/agents/generate-campaign
04LangGraph executes 8 agents
Athena → Atlas → Sophia → Mercury → Nova → Darwin → Apollo → Sentinel (interrupt)
Agent Service · campaign_graph
05Agent returns proposal + agent trace
Includes reasoning, confidence scores, agent decisions, predicted outcomes
Agent Service → App Service (HTTP response)
06App stores decisions & creates campaign
Persists agent decisions in system.agent_decisions, creates campaign + segment + variants
App Service · PostgreSQL
07Approval request created + Telegram sent
Creates approval_request record and sends notification via Telegram bot
App Service · approval_requests + Telegram API
01Approval request created
Status set to 'pending', linked to campaign_id in crm.approval_requests
App Service · crm.approval_requests
02Telegram notification sent
Formatted proposal with Approve/Reject inline buttons
App Service · Telegram Bot API
03Marketer reviews proposal
Reviews via Dashboard (GET /api/v1/approvals) or Telegram inline buttons
App Service · UI / Telegram
04Marketer approves or rejects
POST /api/v1/approvals/{id}/respond with action 'approve' or 'reject'
App Service · POST /api/v1/approvals/{id}/respond
05Approval status updated
Both approval_request.status and campaign.approval_status updated to 'approved'/'rejected'
App Service · PostgreSQL transaction
06Campaign ready for launch
Only approved campaigns can be launched (enforced at API layer)
App Service · Authorization check
01Marketer triggers launch
POST /api/v1/campaigns/{id}/launch
App Service
02App verifies approval status
Rejects if campaign.approval_status != 'approved'
App Service · Guard check
03App resolves segment customers
Evaluates segment criteria to get list of target customer IDs
App Service · crm.segments + crm.customers
04App calls Communication Service (HTTP sync)
POST /api/v1/campaigns/dispatch with campaign_id, customer_ids, channel, variant
HTTP · App → Communication Service
05Communication creates records + queue jobs
Inserts communications rows + queue_jobs in communications schema
Communication Service · PostgreSQL
06App asynchronously triggers lifecycle sim
Fires asyncio.create_task(simulate_campaign_lifecycle) — returns immediately
App Service · Async fire-and-forget
01Queue processor picks up job
Job status transitions: queued → processing → completed
Communication Service · QueueProcessor
02Delivery lifecycle simulation starts
Probabilistic stage progression with realistic time delays
Communication Service · simulate_lifecycle()
03Stage: sent
98% probability · delay 1-10s · communication.status = 'sent'
Communication Service · Append-only event
04Stage: delivered
95% probability · delay 5-60s · communication.status = 'delivered'
Communication Service · Append-only event
05Stage: opened
65% probability · delay 5-120min · communication.status = 'opened'
Communication Service · Append-only event
06Stage: clicked
25% probability · delay 30s-10min · communication.status = 'clicked'
Communication Service · Append-only event
07Stage: converted (or failed)
10% conversion · delay 1-48h · OR failure with reason recorded
Communication Service · Terminal state
08Callback sent to App Service for each stage
POST /api/v1/callbacks/events with communication_id, campaign_id, event_type
HTTP · Communication → App Service
01Comm sends callback event
Sends event_type + metadata to App Service's /callbacks/events endpoint
Communication Service · HTTP POST
02App receives callback
Receives event and routes to update_campaign_performance_from_callback()
App Service · POST /api/v1/callbacks/events
03Increments campaign performance counters
sent_count++, delivered_count++, open_count++, etc. based on event_type
App Service · analytics.campaign_performance
04Recalculates derived rates
delivery_rate, open_rate, click_through_rate, conversion_rate recomputed
App Service · Inline computation
05Dashboard cache invalidated
Next dashboard request recomputes global stats with fresh data
App Service · Redis/KV cache invalidation
06Apollo agent insights available
Performance data feeds AI recommendations for next campaign
App Service · Via Agent Service on next generation
